# XTX - 局域网聊天工具需求文档

## 项目概述

XTX 是一个无服务端的局域网聊天工具，类似飞秋(FeiQ)。用户无需注册、无需添加好友，通过局域网自动发现在线用户并直接聊天。使用 Go 语言开发，提供桌面 GUI 界面，支持多架构交叉编译。

## 核心特性

### 1. 用户发现

- 启动时通过 UDP 广播通知上线
- 定期发送心跳广播维持在线状态
- 退出时发送下线广播
- 自动维护在线用户列表（昵称、IP、端口、上线时间）
- 超时未收到心跳的用户自动标记为离线

### 2. 单聊

- 从在线用户列表选择用户发起聊天
- 支持发送文字消息和图片消息
- 点对点通信，消息通过 TCP 直连发送
- 对方不在线时提示无法发送

### 3. 群聊

- 任意用户可创建群聊，选择在线用户拉入
- 群信息（群名、成员列表）通过 UDP 广播同步给所有成员
- 群消息通过 TCP 逐一发送给每个在线成员（发送者负责分发）
- 成员上线时自动同步已加入的群列表
- 群创建者可修改群名；任何成员可退出群聊

### 4. 本地存储

- 聊天记录按会话（单聊/群聊）分别存储
- 接收的图片保存到本地目录
- 使用 SQLite 存储聊天记录和群信息
- 启动时加载历史记录

### 5. 用户设置

- 可设置本地昵称（默认取主机名）
- 可配置监听端口（默认固定端口）

## 通信协议设计

### UDP 广播（用户发现 + 群同步）

端口：`9527`

消息类型：
| 类型 | 说明 |
|------|------|
| `ONLINE` | 上线通知，携带昵称和TCP端口 |
| `HEARTBEAT` | 心跳包，每30秒一次 |
| `OFFLINE` | 下线通知 |
| `GROUP_CREATE` | 创建群聊，携带群ID、群名、成员列表 |
| `GROUP_UPDATE` | 更新群信息（改名、加人） |
| `GROUP_QUIT` | 成员退出群聊 |
| `GROUP_SYNC` | 新上线用户请求同步群列表 |

### TCP 直连（消息传输）

每个客户端监听一个 TCP 端口（默认 `9528`）接收消息。

消息格式使用 JSON：
```json
{
  "type": "text|image",
  "scope": "private|group",
  "group_id": "",
  "from": "昵称",
  "from_ip": "192.168.1.10",
  "timestamp": 1234567890,
  "content": "文本内容或base64图片数据",
  "filename": "image.png"
}
```

- `scope` 为 `private` 时为单聊，`group` 时为群聊
- 图片传输：小图（<2MB）使用 base64 内嵌；大图通过独立 TCP 连接传输文件流

## 技术方案

### 语言与依赖

- Go 1.22+
- GUI 框架：[Fyne](https://fyne.io/)（纯 Go 跨平台 GUI，无 CGO 依赖于 Linux/macOS，Windows 原生支持）
- 存储：SQLite（modernc.org/sqlite，纯 Go 实现，方便交叉编译）

### 项目结构

```
xtx/
├── main.go              # 入口
├── internal/
│   ├── discovery/       # UDP广播发现模块
│   ├── chat/            # TCP消息收发模块
│   ├── storage/         # SQLite本地存储
│   ├── model/           # 数据模型（User, Message, Group）
│   └── ui/              # GUI界面
│       ├── app.go       # 主窗口布局
│       ├── chat_view.go # 聊天面板
│       ├── user_list.go # 用户/群列表面板
│       └── image.go     # 图片预览与发送
├── assets/              # 图标等资源文件
├── Makefile
├── go.mod
└── go.sum
```

### 多架构构建

通过 Makefile 支持交叉编译：

- `darwin/amd64`、`darwin/arm64`
- `linux/amd64`、`linux/arm64`
- `windows/amd64`、`windows/arm64`

### 布局说明

- **左侧面板**：上方为在线用户列表（绿点在线/灰点离线），下方为群聊列表，底部创建群聊按钮
- **右侧上方**：当前会话的聊天记录，图片以缩略图显示可点击放大
- **右侧下方**：消息输入区，附带图片选择按钮和发送按钮
- 点击左侧用户或群聊切换会话

## 约束与边界

- **仅限局域网**：不考虑 NAT 穿透和公网通信
- **无加密**：局域网内信任环境，暂不加密传输
- **无文件传输**：仅支持文字和图片，不支持任意文件
- **无离线消息**：对方不在线时消息不发送，提示用户
- **群聊去中心化**：无群主服务器，群信息各节点本地存储，通过广播最终一致
